15 // .NAME MySuperquadricSource - create a polygonal superquadric centered
16 // at the origin
17 // .SECTION Description
18 // MySuperquadricSource creates a superquadric (represented by polygons) of
19 // specified size centered at the origin. The alignment of the axis of the
20 // superquadric along one of the global axes can be specified. The resolution
21 // (polygonal discretization)
22 // in both the latitude (phi) and longitude (theta) directions can be
23 // specified. Roundness parameters (PhiRoundness and ThetaRoundness) control
24 // the shape of the superquadric. The Toroidal boolean controls whether
25 // a toroidal superquadric is produced. If so, the Thickness parameter
26 // controls the thickness of the toroid: 0 is the thinnest allowable
27 // toroid, and 1 has a minimum sized hole. The Scale parameters allow
28 // the superquadric to be scaled in x, y, and z (normal vectors are correctly
29 // generated in any case). The Size parameter controls size of the
30 // superquadric.
31 //
32 // This code is based on "Rigid physically based superquadrics", A. H. Barr,
33 // in "Graphics Gems III", David Kirk, ed., Academic Press, 1992.
34 //
35 // .SECTION Caveats
36 // Resolution means the number of latitude or longitude lines for a complete
37 // superquadric. The resolution parameters are rounded to the nearest 4
38 // in phi and 8 in theta.
39 //
40 // Texture coordinates are not equally distributed around all superquadrics.
41 //
42 // The Size and Thickness parameters control coefficients of superquadric
43 // generation, and may do not exactly describe the size of the superquadric.
44 //
